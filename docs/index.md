# Python Learning Journal

Welcome. This site is a record of my journey learning Python: the code I've
studied, written, and broken, alongside notes on what I figured out along the
way.

## How it's organised

The site has two layers, and they serve different purposes.

**Learning Notes** are narrative. They're what I was thinking, what confused
me, what finally clicked. Written roughly in the order I learned things.
Start here if you want to follow the journey.

**API Reference** is generated automatically from the docstrings in the code
itself. Each module, function, and class gets its own page. Start here if you
want to see what the code actually does, or to look something up.

## How to read the code

Every module starts with a top-of-file docstring explaining *why* I wrote it
and what I was trying to understand. Functions have Google-style docstrings
with arguments, return values, and examples. If a choice looks strange, the
reason is usually in a comment right next to it.

## How this site is built

Source code is in [the repo on GitHub](https://github.com/samirinux/python-learning-docs).
The site is generated by [MkDocs](https://www.mkdocs.org/) with the
[Material theme](https://squidfunk.github.io/mkdocs-material/), and API pages
come from docstrings via [mkdocstrings](https://mkdocstrings.github.io/).
Every push to `main` redeploys automatically.
